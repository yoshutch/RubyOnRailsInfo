Ruby on Rails

Ruby on Rails is an open-source web application framework that focuses on convention rather than configuration.

Ruby on Rails uses the Ruby programming language in connection with an SQLite3 database. Rails makes assumptions about what the developer needs to get his/her web application up and running. Therefore the development of the application requires less configuration and gives the developer more time to code the application rather than worry about the framework.

There are two main philosophies that Ruby on Rails attempts to emphasize:

  • DRY - "Don't Repeat Yourself." Repetitive code is bad
  • Convention Over Configuration - Rails makes assumptions about what you want rather than requiring lots of configuration.

General Layout of Rails

Through the command line you can easily create a Rails project. The Rails engine automatically creates the folders and files needed. The general layout or structure of a Rails project helps us understand how Rails works and how similar it is to other MVC frameworks.

  • /app
    • /controller
    • /helpers
    • /models
    • /views
      • /layouts
  • /bin
  • /config
  • /db
  • /lib
  • /log
  • /public
  • /test
  • /tmp
  • /vendor

The main folder is "/app" which contains most of the Ruby code as well as the views and templates. Within in the "/app" folder are the three categories of the Model View Controller (MVC) methodology. The "/app/helpers" folder contains files that assist the model and controller classes. The "/app/views/layouts" folder includes templates that are frequently used in the views.

Despite the fact that Rails focuses on convention rather than configuration it still needs some configuration files, thus the "/config" folder. The "/config" folder also includes the routes file which specifies to Rails which controller is to serve which request to which view.

Some other notable folders and files include: "/public" and "/db". The "/public" folder includes static files viewable to the world, like stylesheets and javascript files. The "/db" folder contains your current database schema where you can manage the database and SQL scripts.

Getting Started with Rails

1 Install Required Software

  • Ruby
  • RubyGems packaging system
  • SQLite3 Database

2 Installing Rails

  1. Make sure that Ruby has been properly installed:
    $ ruby -v
    if it says something like ruby 1.9.3xxx, then we're ready to continue
  2. In order to install Ruby Rails we use the Gem install command:
    $ gem install rails
    OR
    You can use Rails Installer for Windows or Mac. If you have OS X Mavericks or later, simply install rails with: gem install rails
  3. Verify Rails was installed properly:
    $ rails --version
    if it says something like Rails 4.0.0, then we're ready to continue
  4. Create a Rails application:
    $ rails new blog
  5. Run the Rails web server:
    $ rails server
  6. And we're ready to go!
